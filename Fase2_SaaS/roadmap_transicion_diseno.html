Hoja de Ruta: Evolución Web

De un prototipo Streamlit a una Plataforma SaaS de Alto Rendimiento

🏗️ Proceso de Reemplazo: Paso a Paso

Fase 1: Coexistencia Silenciosa

Mientras usás el portal actual para trabajar, nosotros construimos la "página web" (Next.js) y la API en una carpeta separada. No hay riesgo de rotura.

Fase 2: El "Puente" Docker

Conectamos tu lógica actual (el Agente y RAG) a la nueva API dentro de contenedores Docker. El sistema ya puede trabajar en el fondo.

Fase 3: El Gran Switch

Apagamos Streamlit y encendemos la nueva web premium vinculada a la base de datos PostgreSQL. Migramos tu historial automáticamente.

🔄 ¿Qué piezas reemplazamos y por qué?

Componente Actual Pieza Nueva (Fase 2) Mejora Clave
Streamlit UI Next.js + Tailwind Estética premium, animaciones y control absoluto del diseño.
Procesamiento Bloqueante Celery + Redis workers Podés cerrar la web; la auditoría sigue corriendo sola por detrás.
Base SQLite PostgreSQL (Docker) Soporta múltiples usuarios y auditorías masivas sin errores.
st.file_uploader Drag & Drop Custom Validación instantánea y feedback visual de carga.

📊 ¿Podemos diseñar el Dashboard hoy mismo?

Sí. El diseño de la interfaz en Next.js es independiente del backend. Podemos empezar a maquetar el Dashboard con "datos ficticios" para que veas cómo va a quedar, y luego le "enchufamos" el motor del Agente que ya tenemos funcionando.

Esto nos permite iterar la belleza visual del sitio sin tocar el código fiscal que tanto nos costó estabilizar.
