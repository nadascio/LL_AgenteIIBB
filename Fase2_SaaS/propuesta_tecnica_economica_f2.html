Propuesta IIBB Fase 2

Arquitectura Web Profesional & Pricing

v1.2.0

Esta propuesta detalla la transición del actual portal de auditoría hacia una plataforma desacoplada (Frontend + API) con procesamiento asíncrono y despliegue en contenedores.

1. Arquitectura de Sistemas

🖥️ Frontend (Interfaz)

  • Next.js / React (Moderno y Rápido)
  • Dashboard de Auditorías Interactivo
  • Polling de progreso en tiempo real
  • Acceso seguro (Login/Auth)

⚙️ Backend (Cerebro)

  • API Rest con FastAPI (Python)
  • Task Queue con Redis + Celery
  • Base de Datos PostgreSQL (Relacional)
  • Integración Docker Compose

3. Detalle Técnico de Tecnologías

⚡ FastAPI (El Motor)

Es el estándar moderno para APIs en Python. Es gratuito (Open Source).

  • ¿Por qué?: Es hasta 10 veces más rápido que Flask (el viejo estándar).
  • Ventaja: Permite que el Agente trabaje sin "congelar" la web.
  • Costos: US$ 0 por software.

🐘 PostgreSQL (La Memoria)

La base de datos más robusta del mundo. Es gratuita (Open Source).

  • ¿Por qué?: Soporta miles de auditorías simultáneas sin corromperse.
  • Ventaja: Se integra perfecto con Python y SQL.
  • Costos: US$ 0 por software.

¿Dónde "vive" la página web?

Tenés dos opciones para guardar la interfaz (el Dashboard):

  • Opción A (Sugerida): Dentro del mismo contenedor Docker. No tiene costo extra y simplifica todo el sistema en una sola "caja".
  • Opción B (Nube Vercel): Se hospeda por separado en una nube especializada. Es gratis para el volumen de analistas que manejará el portal.

4. Seguridad y API Keys

Las API Keys de los proveedores (Google/OpenAI) se manejarán de forma "Virtualizada" a través de Docker y Secret Managers. Nunca se guardarán en el código.

  • Acceso Controlado: Solo IPs autorizadas o usuarios con Token pueden llamar a la API.
  • Virtualización: Al usar Docker, la app puede moverse entre servidores sin exponer credenciales.
